# Codex 提示词模板

这些句子可以直接复制给 Codex，再把 `{}` 里的内容换掉。

## 1. 新功能

```text
请在当前项目里新增一个功能：{功能名}。
目标是：{你想解决的问题}。
请先查看项目结构，给出简短计划，再开始修改。
改完后请告诉我每个文件做了什么。
```

## 2. 改样式

```text
请把当前页面的视觉风格调整为：{风格描述}。
要求保留现有内容结构，只优化布局、颜色、字号和响应式表现。
请尽量让页面看起来更清晰、更有层次。
```

## 3. 修 bug

```text
我遇到一个问题：
现象：{现象}
报错：{报错}
复现步骤：{步骤}
请先帮我分析可能原因，再给出最小修改。
```

## 4. 解释代码

```text
请用适合 0 基础的方式解释这段代码：
1. 每个文件是做什么的
2. 关键变量和函数是什么意思
3. 我应该先看哪几处
4. 如果我要改它，最安全的入口在哪里
```

## 5. 拆想法

```text
请把这个想法拆成可执行的小步骤：
想法：{想法}
请输出：
1. 目标
2. 用户
3. 输入
4. 输出
5. 最小可行版本
6. 第一批任务
```
