# Codex 实验室

这是一个给 0 基础学习者准备的 Codex 起步项目。它的目标很简单：

1. 先学会把想法拆小。
2. 先做出一个能打开、能修改、能看懂的个人网站。
3. 再把同一套方法，继续用到小工具和 AI 应用里。

## 怎么打开

最简单的方式是直接双击 `index.html`。

如果你想直接打开小工具页面，也可以双击 `codex-prompts.html`。

如果你想用本地服务打开，也可以在这个目录运行：

```powershell
python -m http.server 8000
```

然后访问 `http://localhost:8000`。

## 文件说明

- `index.html`：页面结构和文案。
- `style.css`：颜色、布局、响应式和动效。
- `script.js`：导航高亮、滚动显现、复制提示词。
- `README.md`：项目说明和学习入口。
- `IDEAS.md`：以后每个想法都按同一种方式记录。
- `PROMPTS.md`：你以后和 Codex 对话时的提示词模板。
- `codex-prompts.html`：独立的 Codex 提问模板工具页面。
- `tool.css`：小工具页面的补充样式。
- `codex-prompts.js`：小工具页面的交互和本地保存逻辑。

## 你可以先改什么

- 标题：改 `index.html` 里的首页大标题。
- 颜色：改 `style.css` 里的 `:root` 变量。
- 项目卡片：改 `index.html` 里的“项目展示区”。
- 联系方式：把邮箱、GitHub 等占位内容换成你自己的。
- 小工具：打开 `codex-prompts.html`，试着填写一个真实需求并复制生成结果。

## 你以后怎么用 Codex

推荐你每次都按这个顺序说：

1. 先说目标。
2. 再说限制。
3. 让 Codex 先看项目。
4. 一次只改一个小功能。
5. 改完让它解释给你听。

## 一个很好的起点

如果你现在不知道下一步做什么，就先做这三件事：

1. 打开页面，看看整体结构。
2. 改一个标题。
3. 改一个颜色。

你会很快开始有“我真的能改这个项目”的感觉。
