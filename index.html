目标

先学会把“我想做一个东西”变成“我知道下一步该做什么”。

边做边学

你现在应该怎么学 Codex

01

先说目标

不要一上来就说技术名词，先讲你想解决什么问题。

02

一次一小步

每次只做一个功能，容易看懂，也方便你发现哪里出问题。

03

让 Codex 解释

改完以后要求它用新手能懂的话解释改了什么、为什么这么改。

04

带着报错来

出错时把报错、截图、现象一起给它，效率会高很多。

可复用模板

把下面这些话，直接发给 Codex

加功能

请在当前项目里新增一个功能：{功能名}。
目标是：{你想解决的问题}。
请先查看项目结构，给出简短计划，再开始修改。
改完后请告诉我每个文件做了什么。

改样式

请把当前页面的视觉风格调整为：{风格描述}。
要求保留现有内容结构，只优化布局、颜色、字号和响应式表现。
请尽量让页面看起来更清晰、更有层次。

修 bug

我遇到一个问题：
现象：{现象}
报错：{报错}
复现步骤：{步骤}
请先帮我分析可能原因，再给出最小修改。

解释代码

请用适合 0 基础的方式解释这段代码：
1. 每个文件是做什么的
2. 关键变量和函数是什么意思
3. 我应该先看哪几处
4. 如果我要改它，最安全的入口在哪里
